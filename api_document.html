FrankSx API Documentation v3.2

Welcome to the comprehensive API reference. This documentation covers all endpoints, authentication methods, and best practices for integrating with our platform.

Authentication

All API requests require authentication using API keys. Include your key in the Authorization header:

Authorization: Bearer sk-api-xxxxxxxxxxxxxxxx

Endpoints

GET /api/v3/users

Retrieve a list of users. Supports pagination, filtering, and sorting.

{
  "users": [
    {"id": 1, "name": "Alice", "email": "alice@example.com"},
    {"id": 2, "name": "Bob", "email": "bob@example.com"}
  ],
  "total": 150,
  "page": 1,
  "per_page": 20
}
POST /api/v3/users

Create a new user. Requires admin privileges.

{
  "name": "Charlie",
  "email": "charlie@example.com",
  "role": "developer"
}
PUT /api/v3/users/{id}

Update an existing user.

DELETE /api/v3/users/{id}

Delete a user. This action is irreversible.

Rate Limiting

API requests are limited to 1000 requests per hour per API key. Exceeding this limit will result in a 429 Too Many Requests response.

Error Handling

All errors follow the standard JSON error format:

{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "The 'email' field must be a valid email address.",
    "status": 400
  }
}

Webhooks

Configure webhooks to receive real-time notifications for events such as user creation, updates, and deletions.

POST https://your-domain.com/webhook
Content-Type: application/json

{
  "event": "user.created",
  "data": {"id": 123, "name": "Alice"},
  "timestamp": "2026-05-23T10:00:00Z"
}

SDK Examples

JavaScript

const api = new APIClient({ apiKey: 'sk-api-xxxxxxxxxxxxxxxx' });
const users = await api.users.list({ page: 1, per_page: 20 });
console.log(users);

Python

import requests

headers = {"Authorization": "Bearer sk-api-xxxxxxxxxxxxxxxx"}
response = requests.get("https://api.example.com/v3/users", headers=headers)
data = response.json()
print(data)

cURL

curl -H "Authorization: Bearer sk-api-xxxxxxxxxxxxxxxx"      https://api.example.com/v3/users

Best Practices

  • Store API keys securely and never commit them to version control
  • Use environment variables for sensitive configuration
  • Implement exponential backoff for rate limit handling
  • Validate all input data before sending to the API
  • Monitor webhook delivery status and implement retry logic
